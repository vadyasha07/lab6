# Лабораторна Робота №6: Node.js бекенд-сервіс

## Варіант 5

| Параметр | Значення |
|----------|----------|
| **Фреймворк** | Fastify |
| **Відправка пошти** | Brevo API (REST) |
| **Тунелювання** | Pinggy |

## Структура проєкту

```
lab6-server/
├── public/
│   ├── index.html      # Головна сторінка сайту
│   ├── style.css       # Стилі
│   └── script.js       # Frontend логіка (fetch)
├── server.js           # Головний файл сервера (Fastify)
├── .env                # Змінні середовища (конфігурація)
├── .gitignore          # Ігнорування файлів для Git
└── package.json        # Залежності проєкту
```

## Інструкція з запуску

### 1. Встановлення залежностей

```bash
cd lab6-server
npm install
```

### 2. Налаштування Brevo API

1. Зареєструйтесь на [brevo.com](https://www.brevo.com)
2. Перейдіть в **SMTP & API** → **API Keys**
3. Натисніть **Create a new API key**
4. Скопіюйте ключ
5. Вставте його у файл `.env` замість `your-brevo-api-key-here`

### 3. Налаштування email отримувача

У файлі `.env` замініть `your-email@example.com` на вашу реальну пошту.

### 4. Запуск сервера

```bash
node server.js
```

Сервер запуститься на `http://localhost:3000`

### 5. Тунелювання через Pinggy

1. Відкрийте **нове** вікно терміналу (не зупиняючи сервер)
2. Виконайте команду:

```bash
ssh -p 443 -R0:localhost:3000 a.pinggy.io
```

3. Pinggy згенерує публічний URL (наприклад, `https://xxxxx.a.pinggy.io`)
4. Надайте це посилання викладачу для перевірки
